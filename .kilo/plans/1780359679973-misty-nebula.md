# Plan: Show Desktop Names in Grid View (Super+G)

## Goal
Add desktop name labels below each desktop thumbnail in the KDE Plasma 6 Overview Grid View (activated by Super+G / Meta+G). Labels must be dynamic — they update automatically when desktop names change or desktops are added/removed.

## Current State
- The Grid View is part of the **Overview** builtin KWin effect
- Source QML: `kde-kwin/src/plugins/overview/qml/Main.qml`
- Each desktop is rendered in the `allDesktopHeaps` Repeater as `mainBackground` items
- Desktop names are available via `desktop.name` (QML property with `nameChanged` signal)
- Grid visibility is controlled by `gridVal` (0 = hidden, 1 = fully visible)
- kwin 6.6.5 is installed from system packages

## Implementation Steps

### 1. Modify `Main.qml` — Add Desktop Name Labels

In `kde-kwin/src/plugins/overview/qml/Main.qml`, inside each `mainBackground` item (the delegate of `allDesktopHeaps` Repeater), add a `PC3.Label` element:

- **Text**: Bind to `desktop.name` (auto-updates via `nameChanged`)
- **Position**: Anchor to bottom of `mainBackground`, centered horizontally, with margin below the desktop preview
- **Visibility**: Only show when `gridVal > 0` (grid view mode)
- **Styling**: 
  - Use `Kirigami.Theme.textColor` for text color
  - Add a subtle rounded rectangle background with `Kirigami.Theme.backgroundColor` at ~70% opacity for readability against varying wallpapers
  - Center-align text with `Text.AlignHCenter`
  - Elide long names with `Text.ElideRight`
  - Use `Kirigami.Units.gridUnit` for font size / padding

Placement: Add the label as a sibling to `backgroundArea` and `WindowHeap`, anchored to the bottom of `mainBackground` with appropriate padding.

### 2. Verify No Config Option Needed (Decision Point)

The user asked to simply show names — no toggle requested. If a toggle is desired later, we can add a `ShowDesktopNames` entry to `overviewconfig.kcfg` and wire it up. For now, keep it always-visible in grid mode.

### 3. Build and Install

The Overview effect is a **builtin KWin effect** compiled via CMake:

```cmake
kwin_add_builtin_effect(overview ${overview_SOURCES})
qt6_add_qml_module(overview ...)
```

Options for building:

**Option A: Build full kwin from source**
- Set up build directory for `kde-kwin/` submodule
- Build with `-DBUILD_TESTING=OFF` to save time
- Install with `sudo cmake --install build/`

**Option B: Override QML files (if possible)**
- Qt6 loads filesystem QML over compiled resources if placed in the QML import path
- Test by placing modified `Main.qml` in the appropriate `org/kde/kwin/overview/` path under Qt's QML import directories
- Verify with `qmlimportscanner` or by running `kwin_wayland --replace`

**Decision needed**: Does the user already build kwin from source, or should we set up a build environment?

### 4. Test

- Press Super+G to open Grid View
- Verify labels appear below each desktop
- Verify labels update when renaming desktops (right-click desktop name in Overview)
- Verify labels appear/disappear correctly during gesture transitions
- Verify with 2, 3, 4+ desktops

## Risks & Considerations

1. **Building kwin is heavy** — requires KDE Frameworks 6, Qt6, and other dependencies. If the user doesn't already build from source, we need to set up the build environment.
2. **QML positioning** — The `mainBackground` items use complex transforms (scale, translate) for grid layout. The label must be positioned relative to the *visual* bottom of the desktop preview, not the raw coordinates.
3. **Readability** — Desktop wallpapers vary wildly. A semi-transparent background behind the text is essential.
4. **Translations** — No new translatable strings needed since we're just displaying the desktop name.
