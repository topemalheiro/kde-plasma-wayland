# Fix Hubstaff Crash-Restart Loop on KDE Plasma Wayland

## Problem
Hubstaff opens, stays visible for ~3 seconds, crashes with exit code 255, and its built-in watchdog immediately restarts it — creating an infinite open/close loop.

## Root Cause
KDE Plasma's regional format settings (`~/.config/plasma-localerc`) set `LC_TIME=pt_PT.UTF-8`, but the `pt_PT.UTF-8` locale is **not generated** on the system. The locale archive only contains `C`, `C.utf8`, `en_US.utf8`, and `POSIX`.

This causes:
1. `locale` to fail with: `Cannot set LC_ALL to default locale: No such file or directory`
2. Hubstaff to crash on startup with:
   ```
   Default system locale is unsupported. Either misconfiguration or invalid LANG envvar.
   Forcing C locale failed. Probably the app will crash later on.
   Locale not supported by C library.
   ```
3. Hubstaff's internal watchdog detects the crash and auto-restarts the child process after a short delay.

## Plan

### 1. Generate the Missing `pt_PT.UTF-8` Locale
- Uncomment `pt_PT.UTF-8 UTF-8` in `/etc/locale.gen` (currently commented out; `en_US.UTF-8` is already enabled)
- Run `sudo locale-gen` to rebuild the locale archive
- Verify with `locale` — the "Cannot set LC_ALL" error should disappear

### 2. Verify Hubstaff Launches Successfully
- Launch Hubstaff manually and check `journalctl --user -f` for any remaining locale-related crashes
- Confirm the watchdog does not enter a restart loop

### 3. Document the Fix
- Add a new entry to `KDE-Customization-TODO.md` under "Issues & Fixes" documenting:
  - The locale mismatch between Plasma settings and system locale archive
  - The exact commands used (`sudo locale-gen`)
  - Files modified (`/etc/locale.gen`)
  - How to deactivate (comment out the line in `/etc/locale.gen` and re-run `locale-gen`)

### 4. Fallback (Only if Locale Fix Is Insufficient)
If Hubstaff still crashes after the locale is fixed (e.g. due to remaining Wayland/X11 compatibility issues):
- Modify `~/.local/share/applications/netsoft-com.netsoft.hubstaff.desktop`
- Prepend `env LANG=C.utf8` to the `Exec` line to force a known-good locale for Hubstaff specifically
